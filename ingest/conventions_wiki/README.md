# Python Engineering Style Guide

Welcome to the example internal Python style guide for **Northstar Labs**, an imaginary software development company.

This mini wiki is intended for internal use and is organized into focused documents:

- `01-principles.md` — Core engineering principles
- `02-formatting-and-structure.md` — Formatting, layout, imports, and file organization
- `03-naming.md` — Naming conventions
- `04-functions-and-classes.md` — Guidance for functions, classes, and modules
- `05-types-and-docstrings.md` — Type hints and docstring standards
- `06-errors-logging-and-testing.md` — Error handling, logging, and testing guidance
- `07-tooling-and-review.md` — Tooling, linting, formatting, and code review expectations
- `08-examples.md` — Good and bad examples

## Scope

These standards apply to:

- Production Python services
- Internal tools and scripts that may be reused
- Test code, with pragmatic exceptions
- Automation and data-processing code maintained by Engineering

## Default rule

When this guide conflicts with an existing repository convention, follow the repository convention first unless the team has agreed to migrate it.
